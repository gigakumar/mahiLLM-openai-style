Announcing OnDeviceAI

OnDeviceAI is our local-first automation daemon with an embedded control panel for maintainers who live in open-source. Run TinyLlama-class intelligence on your laptop, wire it into your GitHub rituals, and keep contributor workflows humming without a cloud hand-off.

Why we built it

Coordinating open-source projects still means juggling pull requests, changelogs, releases, docs updates, and community questions across half a dozen tools. We wanted a co-pilot that respects privacy, works offline, and understands the rituals of maintainers. OnDeviceAI pairs a Rust automation core with Python planners and our TinyLlama deployments to do exactly that.

What OnDeviceAI can automate today

  • Tag, prioritize, and cluster new GitHub issues based on labels, sentiment, and maintainer load.
  • Draft release notes, changelog entries, and upgrade guides straight from merged pull requests.
  • Surface stale discussions, apply triage labels, and nudge contributors before community momentum drops.
  • Routinely sync docs or blog updates by generating PRs against your documentation repos.

Under the hood

  • Rust daemon orchestrating connectors, schedulers, and a secure task queue.
  • TinyLlama and MahiLLM distilled models running locally via MLX or Metal acceleration.
  • PyWebview desktop control panel with live daemon telemetry and plan approval UX.
  • Signed automation recipes and encrypted vault storage to keep secrets on-device.

What's next

  • Expanding automation modes beyond open-source into community support and docs sprints.
  • Adding programmable checkpoints so teams can version-control their automation recipes.
  • Packaging a starter marketplace of maintainer templates and integration bundles.

We're piloting OnDeviceAI with a handful of maintainers today. If you want in, drop us a line at hello@mahillm.ai or spin up the daemon inside the On-device Assistant workspace.
